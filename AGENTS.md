# TokenSaver Repository Instructions

对本仓库进行任何代码、测试、配置、文件或文档变更前，必须先完整阅读并遵守：

- `docs/governance/DEVELOPMENT_PRINCIPLES.md`
- `docs/governance/README.md`

强制顺序：先建立 PRD，再开发，再执行测试 SOP，最后更新测试记录和开发记录。缺少任一步骤时，不得声称开发完成。

`docs/governance/DEVELOPMENT_PRINCIPLES.md` 是受控文件。除非用户张涛在当前任务中对原则变更作出明确、直接授权，否则禁止修改该文件及其完整性基线。一般性的“继续开发”“更新文档”“修复测试”不构成原则变更授权。
