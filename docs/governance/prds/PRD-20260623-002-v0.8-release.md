`PRD-20260623-001-onboarding-pipeline-handoff.md` 已完成实现和本地测试，但变更尚未提交、合并、打标签或发布到 PyPI。本次新增兼容的公开 SDK API、诊断规则和输出字段，依据 `VERSION.md` 的 SemVer 规则发布为 `0.8.0`。

目标是以可审计方式完成独立发布分支、版本元数据、构建验证、draft PR、GitHub CI、合并、`v0.8.0` 标签、Trusted Publishing 和 PyPI 隔离安装验证。

- 执行发布前测试、构建、本地 wheel 安装、版本与 demo 验证。

1. 从最新 `origin/main` 创建 `codex/release-v0.8.0` 并携带已审计工作区变更。
2. 更新 0.8.0 版本元数据与正式 changelog 条目。

## 版本与兼容性

- 新版本：`0.8.0`。
- 版本类型：MINOR；新增 `AgentRun.add_handoff`、`handoffs` 可选字段和新 diagnosis code，均为兼容性增加。
- schema 继续为 0.4；`handoffs` 为 additive optional field，旧 trace 保持可读。
- 无新增运行依赖。

## 风险与回退

- PyPI 版本不可覆盖：发布前必须确认 0.8.0 尚不存在，并验证 wheel/sdist 元数据。
- 当前工作区起点位于旧发布审计分支：已确认其 tree 与最新 `origin/main` 一致；仍必须从 `origin/main` 建立新分支并核对 diff。
- CI 失败时停止合并；修复必须保持在本 PRD 范围内并重新执行受影响测试。
- Trusted Publishing 失败时保留标签和构建证据，修复 workflow 后重新 dispatch；不得覆盖 PyPI 文件。
- 使用方升级失败时回退依赖 pin 到 `tokensaver-agent==0.7.0`。
- 标签错误但尚未发布 PyPI 时可以删除并重建；PyPI 发布后不得移动同名标签。
